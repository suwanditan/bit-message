Bit Message is a message that hidden within lines of octet stream and represented as hexadecimal string that have length maximum 149 octets. For 9 octets of the message is a header message and the rest, is its content. Header message contains 1 octet of type, 7 octets of timestamp, and 1 octet of message length. Content message contains maximum 140 octets, and could be packed with 7 bit, 8 bit or 16 bit. 7 bit packed message has length of 160 characters, 8 bit packed message has length of 140 characters, and 16 bit packed message will only has length of 70 characters.

TYPE 1 contains specific flag for specific format identifier

Bit 2-3 : message encoding

Bit 4 : reserved flag message class meaning

Bit 5 : reserved message is compressed or uncompressed

Bit 6-7 : reserved general data coding

Octet 7 TIMEZONE





The Time Zone is GMT format, expressed in quarters of an hour. In the first of the two semi-octets, the first bit represents the algebraic sign of this difference (0 : positive, 1 : negative)

















LENGTH 1 maximum characters allowed is 160, 140 or 70 characters depends on message packed format






































Input data: A hexadecimal string that is bit message.

Output data: The output will be a list contains timestamp, length of message and the message itself.

Example:

assert( checkio( '002080629173140807EDF27C1E3E9701' ) ==
['26 Aug 2002 19:37:41 GMT +20', 7, 'message'] ), "First Test"

assert( checkio('00317050201171820ED3323BDC0ED341C4303DEC3E8700') == 
['05 Jul 2013 02:11:17 GMT +7', 15, 'Selamat Datang!'] ), "Second Test, 7 bit"
    
assert( checkio('000130925161956915C8729E054A82C26D50DA0D7296EFA0EC5BBE06') ==
